# dbskill

dontbesilent 商业诊断工具箱。从 12,307 条推文中提炼方法论，做成 Claude Code skill。

## 数据权威索引

文件位置、数据冲突规则见 `SOURCE_OF_TRUTH.md`。

## 项目结构

- `skills/` — 7 个 SKILL.md（6 个 dbs 工具 + 1 个 chatroom）
- `知识库/` — 原子库 + Skill 知识包 + 高频概念词典
- `chatbot/` — chatbot 部署用的 system prompt（不发 GitHub）
- `plans/` — 内部计划文档（不发 GitHub）

## 工作模式

### 进度更新模式（`/进度`）

用户说 `/进度` 时启动。核心：详细 + 高频。

- 使用进度条：`🚀 进度：▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░░░░░░░ 20% - 正在处理文件 2/12`
- 完成时：`✅ 进度：▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ 100% - 完成！`
- 每个步骤显示：当前操作、具体文件名/路径、中间结果
- 批量操作每 5-10 个文件更新一次

### 少授权模式（`/少授权`）

用户说 `/少授权` 或 `/免打扰` 时启动。核心：最少打断 + 看得懂。

- **始终生效**：每次 Bash 命令前用一句中文解释，格式 `📋 这条命令的作用：[解释]`。只读操作标注「只读」，修改/删除标注「⚠️」
- **触发后生效**：合并命令减少授权次数、优先用免授权工具（Glob/Grep/Read）、任务开始前预告所有命令
