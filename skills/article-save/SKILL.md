# 文章保存与处理流程

## 核心路径

文章存放目录：`D:\100 - 工作\130 - 数字资产\dbskill\文章摘取\`

文件命名规范：`YYYY-MM-DD.md`（日期为处理当天日期）

## 工作流程

1. **读取内容**：接收用户提供的文字稿内容
2. **创建文件**：在目标目录创建 `YYYY-MM-DD.md` 文件
3. **格式化内容**：
   - 标题使用当天日期：`# YYYY-MM-DD 文章记录`
   - 来源（视频/文章/播客等）
   - 核心内容摘要（提取关键观点，保持原文精华）
   - 精彩段落原文摘录（用引用格式）
   - 记录时间戳
4. **同时保存**：同步保存到 workspace `memory/YYYY-MM-DD.md` 作为备份记录
5. **确认完成**：告知用户保存路径和文件名

## 内容组织原则

- **标题**：当天日期 + 内容主题关键词
- **来源**：标注清楚来源类型和名称
- **核心要点**：用简洁的 bullet points 概括视频/文章的主要观点
- **精华摘录**：保留原文中特别有价值、值得回味的段落
- **个人感悟**（如有）：用户分享时附带的想法或联系到其他内容的延伸思考
- **记录时间**：标注处理时间

## 文件模板

```markdown
# YYYY-MM-DD 文章记录

## 来源
[来源类型]：<名称>

## 核心内容
- 要点1
- 要点2
- ...

## 精华摘录
> 原文摘录1
>
> 原文摘录2

## 相关思考（可选）
[用户附带的个人想法]

---
*记录时间：YYYY-MM-DD HH:MM*
```
